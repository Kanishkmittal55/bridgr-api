# bridgr-api

Standalone Bridgr skill-gap HTTP API + SQS worker. Chi router, sqlc + Postgres, MinIO presigns, LocalStack SQS for async analysis pipeline. **Single OpenAPI surface**: tag `bridgr` only (presign stays at `POST /v1/autograph/upload-url` but is generated into the Bridgr server package).

## Run with Docker

From the **parent folder** of this repo (`users/`), so `hassle-go/` is available for the image build:

```bash
docker compose -f bridgr-api/docker-compose.yaml up --build
```

- API: `http://localhost:8080` (header `X-API-KEY: test-all-access-key`)
- Postgres: host port **5433** → container 5432 (avoids clashing with a local 5432)
- MinIO: `9000` / console `9001`
- LocalStack SQS: `4566`, queue `bridgr-skill-gap`

Set `ENV=development` (required by `hassle-go` env resolution). Compose does this for `api` and `worker`.

Compose project name is **`bridgr-api`** (see top-level `name:`), so the default network is **`bridgr-api_default`** — used by `make migration-structure-sql`.

## Makefile (aligned with `users/users/Makefile`)

| Target | Purpose |
|--------|---------|
| `make openapi-generate` | Same pattern as users: Docker image (`docker/openapi/Dockerfile`) with **Go 1.24.12-alpine3.23**, **Redocly CLI**, **oapi-codegen v1.16.3** — bundles `docs/api/index.yml` → `internal/api/open_api/OpenAPISpec.gen.yaml`, then runs every `docs/api/config/*.yaml`. |
| `make generate-sqlc` | Installs **sqlc @ v1.24.0** (via `go install`), runs `sqlc generate -f db/sqlc.yaml`. |
| `make generate` | Currently alias for `generate-sqlc` (matches users `generate` composition you can extend). |
| `make build-local-images` | Builds **`hassle-skip-migrate-local:latest`** from `docker/migration/Dockerfile` (same layout as users). |
| `make create-migration <name>` | Runs `migrate create` in that image (timestamped `.up.sql` / `.down.sql` under `db/migrations/`). Example: `make create-migration add_widget`. |
| `make migration-structure-sql` | **pg_dump** schema dumps for tooling (see below). |

### When to run `make migration-structure-sql`

Run it when:

1. **Postgres is up** on `bridgr-api_default` (e.g. after `make up` or at least `docker compose up -d postgres migrate` so the DB exists **and migrations have been applied**).
2. You want to **refresh** `db/migration-structure.sql` (full `--create` style dump) and **`db/sqlc-structure.sql`** (schema-only for sqlc, matches users’ second `pg_dump`).

**Immediately after**, run:

```bash
make generate-sqlc
```

so `internal/repository/sqlc` matches the live DDL. Commit updated `db/sqlc-structure.sql` when the schema change is intentional.

New SQL migrations: **`make create-migration <name>`**, edit the generated files, bring up **`migrate`**, then **`migration-structure-sql`** → **`generate-sqlc`**.

## OpenAPI layout (`docs/api/`)

| Path | Purpose |
|------|---------|
| `docs/api/index.yml` | Root OpenAPI entry |
| `docs/api/resources/*.yml` | Per-operation fragments |
| `docs/api/schemas/*.yml` | Schemas |
| `docs/api/config/bridgr-server.yaml` | Chi server → `internal/api/bridgr/bridgr.gen.go` |
| `docs/api/config/bridgr-models.yaml` | Models → `pkg/types/bridgr.gen.go` |
| `docs/api/config/client.yaml` | Client + models → `pkg/client/client.gen.go` |
| `internal/api/open_api/OpenAPISpec.gen.yaml` | Redocly bundle (regenerated by `make openapi-generate`) |
| `internal/api/open_api/openapi.go` | Optional `LoadSpec()` embed (same idea as users) |

## Local Go (no Docker)

```bash
export ENV=development
export POSTGRES_HOST=localhost
export POSTGRES_PORT=5433
make generate-sqlc   # after changing queries or db/sqlc-structure.sql
go run ./cmd/api
```
